## چیکار کردیم تا الان

- freellmapi رو امتحان کردیم، پر مشکل بود (باگ داخلی، وابستگی مشکوک به سرور خارجی) → کنار گذاشتیم
- 9Router هم بررسی و رد شد (دور زدن ToS سرویس‌های subscription)
- تصمیم: مستقیم به **Groq** وصل شدیم، بدون پروکسی واسط
- مشکل بزرگ: Node.js پروکسی سیستم رو خودکار نمی‌خونه (برخلاف curl/مرورگر) → با `undici` حلش کردیم
- نتیجه: مونوریپو ساخته شد، `/chat` endpoint روی Fastify زنده و تست‌شده‌ست

## مسیری که برای ساخت مونوریپو رفتیم

اول یک پوشه‌ی روت برای کل پروژه ساختیم و با `pnpm init` بهش شناسنامه دادیم. بعد یک فایل تنظیمات pnpm workspace اضافه کردیم که به pnpm میگه دو تا مسیر `apps/*` و `packages/*` باید به‌عنوان زیرپروژه‌های مستقل ولی مرتبط شناخته بشن.

بعدش turborepo رو به‌عنوان devDependency روی روت پروژه نصب کردیم و یک فایل تنظیمات براش نوشتیم که میگه چه taskهایی (مثل `dev` و `build`) باید بشناسه و چطور کششون کنه.

بعد رفتیم سراغ ساخت اولین اپلیکیشن واقعی: `apps/server`. اونجا هم یک `package.json` مستقل ساختیم و پکیج‌هایی که لازم داشتیم رو نصب کردیم:
- **fastify** برای خود سرور
- **zod** برای اعتبارسنجی ورودی‌ها
- **openai** به‌عنوان SDK ارتباط با LLM (که چون Groq سازگار با فرمت OpenAI هست، همینو استفاده کردیم)
- **dotenv** برای خوندن متغیرهای محیطی از فایل `.env`
- بعداً **undici** هم اضافه شد، مخصوص حل مشکل پروکسی

برای TypeScript هم `typescript`، `tsx` (برای اجرای مستقیم فایل‌های .ts بدون نیاز به build دستی)، و `@types/node` رو به‌عنوان devDependency اضافه کردیم.

یک نکته‌ی جانبی این وسط: pnpm به‌خاطر مسائل امنیتی، اجرای اسکریپت‌های نصب بعضی پکیج‌ها (مثل esbuild که زیرساخت tsx هست) رو به‌صورت پیش‌فرض بلاک می‌کنه، و باید صریحاً تاییدش کنیم تا درست کار کنه.

### تست کردن
اول یک route ساده به اسم `/health` ساختیم که فقط یک پیام "ok" برمی‌گردوند — فقط برای اطمینان از اینکه خود Fastify درست بالا میاد و پورت باز میشه. با `curl` تستش کردیم و جواب گرفتیم.

بعد از اون مطمئن شدیم، رفتیم سراغ ساخت واقعی route اصلی (`/chat`) که به Groq وصل میشه. اون‌جا بود که به مشکل پروکسی برخوردیم (که جدا توضیح دادیم) و بعد از حلش، با یک curl دیگه پیام فارسی فرستادیم و جواب واقعی از مدل گرفتیم — یعنی مسیر کامل از Fastify تا LLM تست و تایید شد.

### ساختار فعلی پروژه

```
ai-docs-assistant/
├── apps/
│   └── server/          → Fastify API (تنها اپ فعلی)
│       └── src/
│           ├── index.ts      → نقطه‌ی ورود، تعریف route ها
│           ├── llm.ts        → کلاینت Groq
│           ├── schemas.ts    → schema های zod
│           └── proxy.ts      → تنظیم پروکسی برای Node
├── packages/             → هنوز خالیه، بعداً پکیج‌های مشترک اینجا میان
├── pnpm-workspace.yaml
├── turbo.json
└── package.json
```

فعلاً فقط `apps/server` ساخته شده؛ `apps/web` (فرانت React) و `packages/*` (schema/db/ai مشترک) هنوز نساختیم — قرار بود قدم‌های بعدی که پیش بریم اضافه بشن.
